Patterns
GuidedStep
One task per screen. Replaces scrollable long-form. Required fields surface before optional ones. Conditional fields appear only when a prior answer triggers them. Completion enforcement feels like natural sequence, not adversarial blocking.

Design rules
Non-negotiable constraints
1
Minimum 44px touch target
Every interactive element — buttons, checkboxes, chips, cards — must be at least 44×44px. Field technicians frequently work with gloves. This is not adjustable.
2
Offline is the default state
Design for offline first. Online is the exception. Never block the technician from completing work due to connectivity. Never show a "No Connection" banner — it implies they should stop. They should not stop.
3
Status colors are semantic, not decorative
Clay = paused/urgent. Sage = active/success. Amber = transit/warning. Terracotta = brand interaction only. Never use these colors for anything outside their designated role.
4
OrderCard shows 3 things at list level
Status, destination, time. That's it. More than 3 data points at the list level creates scanning effort at the moment the technician's working memory is most loaded.
5
Literata is for status moments, not body text
Literata carries authority. Use it for KPI numbers, the pause state hero, screen titles at 24px+, step counters. Never use it for body text, labels, or UI copy — that's Manrope's territory.
6
Signature is a step, not a screen
Signature collection lives inside GuidedStep as step 4 of 4. Never as a separate screen reached via a button. The re-sign affordance is a labeled button below the canvas — not an × on the signature itself.
