## 0. How to resume at home (read this first)

1. **Carry the whole `breno/` folder** to your machine — by `breno-portfolio-snapshot.zip`
   (in this folder, excludes the heavy generated dirs), a USB copy, or git.
   - You do **not** need to copy `portfolio/node_modules/` (434 MB) or
     `portfolio/.next/` (120 MB) — they're regenerated by `npm install` / `npm run build`.
2. Install **Node.js 18.18+** (built/tested on Node 24). On Windows: `winget install OpenJS.NodeJS.LTS`.
3. Run the app:
   ```bash
   cd portfolio
   npm install        # restores node_modules from package-lock.json
   npm run dev        # http://localhost:3000
   ```
4. Open the `breno/` folder in Claude Code and point it at this file
   (`HANDOFF.md`) so the new session has full context. There's also a git history
   (see §6) with a snapshot commit.

This machine is not the owner's — Node here and `node_modules` are Windows-specific;
reinstall fresh at home, don't try to copy them.

## 3. Current state — what's built

### Pages / routes
- **`/` — single-page landing**, the 7 reference mockups in sequence:
  1. **Hero** — "Product designer with a *technical* edge" + 2 CTAs + editorial image placeholder.
  2. **Selected work** — 4-up editorial cards: 1 real case + 3 honest "coming" placeholders.
  3. **Approach** — numbered editorial rows (how I work, 5 items).
  4. **Process** — 6-column grid (Discover → Deliver) with per-item checklists.
  5. **Principles** — numbered rows + Japanese annotations (6 values).
  6. **About** — copy · portrait placeholder · facts sidebar (location, languages, cert).
  7. **Contact** — email / location / live local time, then footer.
- **`/work/service-operations`** — full case study from `case-content-service-operations.md`
  (all 5 blocks + design-system slice specimen + honest "— to add" frames).
- **`/about`** — long-form transition narrative (operations → product design).

### Design system wiring (important)
- `design-system/styles.css` is **imported for real** in `layout.jsx`; every colour,
  font, space, easing is a DS CSS variable. No invented values **except the accent override** (§5).
- The 9 DS components were **rebuilt as clean React** in `src/components/ds/` from the spec
  files. The shipped `_ds_bundle.js` runtime is **not** used.
- **Fonts:** DS declares DM Sans + Cormorant Garamond via `@import` in
  `design-system/tokens/fonts.css`. Bundlers strip nested remote `@import`s, so the
  **same** Google Fonts request is mirrored as a `<link>` in `layout.jsx`. Keep both.

### Motion (all reduced-motion + no-JS safe)
- **Lenis** smooth scroll (`SmoothScroll.jsx`), no parallax.
- **Nav anchors** smooth-scroll to the exact section title, offset 88px below the sticky
  header. Handler runs in **capture phase** so `next/link` doesn't intercept the hash;
  target Y is computed numerically (Lenis element-offset math didn't match the layout).
- **Reveal.jsx** — fade-up; `mask` variant makes headings *rise* from a clip (Awwwards feel).
- **Cascade.jsx** — reveals a block's pieces **in sequence by content type** (number →
  title → description → each list item), e.g. Process "Discover". Snappy (~45–55ms steps).
- **ScrollProgress.jsx** — top clay→slate progress bar + a "NN / 07 · label" section indicator.
- **ToTop.jsx** — back-to-top button (appears after first viewport).
- **Pulsing dot** on the "Available for new work" indicator (CSS `pulse-ring`).

### Accent colour
- Swapped from the DS's **Clay (brown #8A6A4E)** to **Slate Blue #3C4A63** at the user's request.

- **No invented clients, logos, or metrics.** The reference mockups (images 1–7) were used
  for **layout only** — all their text/numbers/logos are fictional and were discarded.

- Voice: direct, mature, sentence case, numbers over words, no buzzwords, no emoji.
- The case study's two personal lines (Trade-offs & next steps · AI-assisted note) are
  Breno's own words, already in place.

## 5. The accent colour — how to change it again

One place. In `portfolio/src/app/globals.css`, near the top, the **Accent override** block
repoints the whole Clay token family (the app loads after the DS, so it wins):

```css
:root {
  --clay:       #3C4A63;                  /* accent — slate blue (was #8A6A4E clay/brown) */
  --clay-dark:  #2E3A50;                  /* hover / pressed */
  --clay-light: #8791A6;                  /* light slate */
  --clay-ring:  rgba(60, 74, 99, 0.18);   /* focus ring */
  --clay-wash:  rgba(60, 74, 99, 0.07);   /* subtle wash / frames */
}
```

Everything (eyebrows, numbers, dots, links, button, progress bar, pulse) uses `var(--clay)`,
so editing these 5 lines re-themes the whole site. Other palettes discussed:
forest green `#46604F`, monochrome ink `#1C1A18`, vermilion `#C44A31` (the DS error red).

## 7. Key decisions log (so the next session knows *why*)

1. Design system is the source of truth — link `styles.css`, rebuild the JSX components,
   don't use the runtime bundle.
2. Mockups = **layout reference only**; content is Breno's, fictional mockup content removed.
3. The 7 images are the **landing page in sequence** → one page with all 7 sections; Case
   and long-form About are separate routes too.
4. Nav must scroll to the **exact section** (capture-phase handler + numeric offset).
5. Reveals should be **granular by content type** (Cascade), "smooth and smart, not turtle".
6. Accent changed **Clay → Slate Blue #3C4A63**.
7. Next was upgraded **15.1.6 → 16.2.9** to clear a security advisory (CVE-2025-66478);
   React on 19.2.
